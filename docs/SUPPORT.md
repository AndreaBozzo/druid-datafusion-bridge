# Support

## Getting Help

### Documentation

- **[README.md](../README.md)** - Project overview and quick start
- **[CONTRIBUTING.md](CONTRIBUTING.md)** - Contribution guidelines
- **[docs/](.)** - Additional documentation and guides

### Asking Questions

1. **Search existing issues** - Your question might already be answered
2. **Check the README and docs** - Common questions are often documented
3. **Open a discussion issue** - Use the `question` label to ask for help

### Reporting Issues

If you've found a bug:

1. **Search existing issues** to avoid duplicates
2. **Provide a clear description** with:
   - Steps to reproduce
   - Expected behavior vs. actual behavior
   - Your Rust version, OS, and environment
   - Relevant error messages or logs
3. **Use the `bug` label** when creating the issue

### Feature Requests

Have an idea for an improvement? 

1. **Check existing issues** to see if it's already been requested
2. **Open an issue with the `enhancement` label**
3. **Describe the use case** and why it would be valuable

## Getting Involved

### Contributing Code

See [CONTRIBUTING.md](CONTRIBUTING.md) for detailed guidelines on:
- Setting up your development environment
- Making changes and submitting PRs
- Code style and testing requirements

### Reviewing PRs

We welcome code reviews from community members! If you have expertise in:
- Binary format parsing
- Rust best practices
- DataFusion integration
- Performance optimization

Please feel free to review and comment on open pull requests.

## Communication Channels

- **GitHub Issues** - For bugs and feature requests
- **GitHub Discussions** - For questions and general discussion
- **GitHub Pull Requests** - For code contributions

## Security Issues

If you discover a security vulnerability, please email the maintainers directly rather than opening a public issue. Please do not post security vulnerabilities on the GitHub issue tracker.
